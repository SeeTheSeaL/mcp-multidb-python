# 数据库连接测试说明

## 配置说明

1. **connections.json**（项目根目录，或由各引用项目通过 `MULTIDB_CONFIG_PATH` 指定）  
   格式示例：
   - URL、user、password，与 Java JDBC 一致
   - 该文件已在 `.gitignore` 中，不会提交到仓库

2. **虚拟环境**  
   项目下已创建 `.venv` 并安装 `jpype1` 和本包。

## 连接测试方式

请通过 **MCP 工具** 在 Cursor 等客户端中测试连接，无需本地脚本：

- **multidb_list_connections**：列出已配置连接，确认连接已加载
- **multidb_list_tables**：传入 `connectionId`，可列举该库下的表
- **multidb_query**：传入 `connectionId` 和 `sql`，可执行只读 SQL（如 `SELECT 1`）

在 Cursor 中配置好本 MCP 并指定对应项目的连接配置文件后，直接使用上述工具即可验证连接是否正常。

## lib/ 驱动

将对应数据库的 **JDBC 驱动 jar** 放入项目根目录的 **`lib/`** 下（如 Oracle：ojdbc8.jar，达梦：DmJdbcDriver18-*.jar）。  
与 Java 项目使用同一批驱动文件即可。
